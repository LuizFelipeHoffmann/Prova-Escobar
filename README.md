# Prova-Escobar
Questão 1 – Factory Method

Usado para permitir a criação de diferentes tipos de relatórios sem alterar o código principal. Facilita extensão e mantém baixo acoplamento.

Questão 2 – Strategy

Usado para trocar dinamicamente o algoritmo de cálculo de risco (agressivo, moderado, conservador) sem modificar o fluxo principal da aplicação.

Questão 3 – Observer

Usado para notificar automaticamente vários módulos (painel, alertas, histórico) sempre que o sensor atualiza seus valores.

Questão 4 – Chain of Responsibility

Usado para montar uma cadeia flexível de verificações antifraude, permitindo barrar a transação em qualquer etapa e adicionar novas validações facilmente.

Questão 5 – Singleton (Thread-Safe)

Usado para garantir uma única instância global do módulo de logging, evitando inconsistências em ambientes multi-thread.
